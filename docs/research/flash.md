# Flash Trade v2 API — integration brief for Throtl (verified live, 2026-06-13)

## 0. The headline finding that changes Throtl's architecture
**Flash Trade v2 ("magic-trade") is itself a MagicBlock Ephemeral Rollup product.** The live `GET https://flashapi.trade/v2/health` returns `"program":"magicblock"`; the user's per-wallet **Basket** PDA is delegated to a dedicated Flash ER validator and all trading txs execute on the ER at ~30–50 ms, gasless, with state auto-committed to Solana L1 **every 10 s** (commit frequency and validator are protocol-fixed server-side; the ER's `getIdentity` returns `FLAshCJGr4SWk23bDVy7yeZecfND8h5Cingy1u2XE6HQ`). Source: https://github.com/flash-trade/examples-v2 (README, pushed 2026-06-12 — "the first public client of V2", built for MagicBlock's Solana Blitz v5 hackathon) and live API probes. Consequence for Throtl: the hysteresis reconciler does **not** need to wait for L1 commit boundaries to settle into Flash — Flash v2 fills are ER-speed. But Flash's ER and your throtl-engine's ER are separate delegation domains: your program still cannot CPI into Flash's basket; the reconciler should call Flash's REST tx-builder and submit to Flash's ER RPC (optionally signed by a session key, see §5).

## 1. Base URLs, auth, rate limits
- **API base:** `https://flashapi.trade` — v2 endpoints under `/v2` (https://docs.flash.trade/flash-trade/flash-trade-protocol/build-on-flash/flash-trade-api/flash-trade-v2/workflows/setupintegration.md). Swagger UI: `https://flashapi.trade/docs`; full OpenAPI vendored at https://github.com/flash-trade/examples-v2/blob/main/openapi.v2.json (75 schemas, 36 v2-relevant paths).
- **ER RPC (trading tx submission):** `https://flash.magicblock.xyz`. **Base RPC (setup/withdrawal):** any Solana mainnet RPC (public one rate-limits). Env names used by the reference client: `FLASH_V2_BASE_URL`, `ER_RPC_URL`, `BASE_RPC_URL` (examples-v2 `.env.example`, `packages/flash-v2/src/network.ts`).
- **Auth: none.** "No API key required. All endpoints are publicly accessible" (v1 endpoints-overview, shared infra). REST rate limit **10 req/s**; WebSocket limits **5 connections/owner → 429**, **10,000 global → 503** (https://docs.flash.trade/.../flash-trade-v1/endpoints-overview.md, .../flash-trade-v2/websocketstreaming.md). CORS is `access-control-allow-origin: *` on both flashapi.trade and flash.magicblock.xyz (verified via curl) — Flutter web can hit both directly.

## 2. Transaction-builder endpoints (all POST, all return base64 **v0 VersionedTransaction**)
Canonical reference: https://docs.flash.trade/flash-trade/flash-trade-protocol/build-on-flash/flash-trade-api/flash-trade-v2/apireference.md. **CORRECTION (live-verified 2026-06-13 — supersedes earlier "partially signed" wording):** transactions come back **UNSIGNED**. There are **no server signatures** today; the server only pre-fills the `recentBlockhash` for the correct chain (ER blockhash for trades, base blockhash for setup/withdraw). The client adds **exactly one** signature (owner, or session signer) and **must not replace the blockhash** — not to preserve a foreign signature (there is none), but because the blockhash is chain-specific and the tx must be submitted to the chain the builder targeted (ER RPC for trades, base RPC for setup/withdrawal). Do **not** write signature-slot-preservation logic; do place the single signature at the index matching the signer's position in `accountKeys` (see the `package:solana` signature-ordering FIXME in flutter.md). Blockhash expires ~45 s; rebuild on expiry.

**Trading (submit to ER RPC):**
- `/v2/transaction-builder/open-position` — `{inputTokenSymbol, outputTokenSymbol, inputAmountUi, leverage, tradeType:"LONG"|"SHORT", orderType?:"MARKET"|"LIMIT", limitPrice?, takeProfit?, stopLoss?, owner?, slippagePercentage?(default "0.5"), signer?, sessionToken?}`. Omit `owner` → **preview-only quote** (`transactionBase64:null`); `inputAmountUi:"0"` → liquidity probe. Opens or **increases** the position. Response `OpenPositionResponseDto`: `newLeverage, newEntryPrice, newLiquidationPrice, entryFee, availableLiquidity, outputAmountUi, youPayUsdUi, youRecieveUsdUi` (typo is real — do not "fix"), plus `err` on compute failure inside HTTP 200.
- `/v2/transaction-builder/close-position` — `{marketSymbol, side, inputUsdUi, withdrawTokenSymbol, owner, ...}`. **`inputUsdUi ≥ 97% of size` (or "0") silently becomes a FULL close** (different on-chain ix) — design the reconciler around this threshold (GOTCHAS.md §4).
- `/v2/transaction-builder/reverse-position` — atomic flip; `side` = current side; fixed **2% haircut** on close proceeds.
- `/v2/transaction-builder/add-collateral`, `remove-collateral` (`depositAmountUi`/`withdrawAmountUsdUi`).
- Triggers: `place-trigger-order`, `place-tp-sl` (atomic TP+SL), `edit-trigger-order` (must resend BOTH price+size), `cancel-trigger-order` (`orderId` 0–4; **255 = cancel all**). Limits: place via open-position `orderType:"LIMIT"`; `edit-limit-order` (0/omit = keep — **opposite semantics** to trigger edit; TP/SL do NOT persist across edits), `cancel-limit-order`.
- Preview (no tx): `/v2/preview/exit-fee`, `/v2/preview/margin`, `/v2/preview/tp-sl` (modes `forward`/`reverse_pnl`/`reverse_roi`, supports inline positions), `/v2/preview/limit-order-fees` — present in openapi.v2.json though absent from the docs quick-reference table.

**Account & funds (submit to base RPC), strict order:** `init-deposit-ledger` → `init-basket` → `delegate-basket` (`{payer, owner}`; commitFrequency/validator ignored, fixed at 10,000 ms / `MAS1Dt9...`) → `deposit-direct` (`{owner, tokenMint, amount}` — **mint pubkey + UI amount**, server resolves ATA/Token-2022) → trade → `request-withdrawal` → (recovery) `execute-withdrawal` with `includeCustodySettlement`. Withdrawals are two-phase: validator crosses a `settlement_receipt` to base in ~30–90 s; premature execute fails `0xbc4 AccountNotInitialized` — poll with unsigned `simulateTransaction({sigVerify:false, replaceRecentBlockhash:true})`. SOL arrives as WSOL (unwrap). The API does NOT enforce lifecycle order; the program does (errors 6018 InvalidBasketState etc.).

**Error channels (three):** trading/preview → HTTP 200 with `err` in body; trigger/limit → HTTP 400 `{error}`; setup/withdrawal → bare 500 with empty body. Full on-chain error table (6000–6111) at https://docs.flash.trade/.../flash-trade-v2/errorreference.md — key ones: 6020 MaxPriceSlippage, 6021/6022 leverage caps, 6034 MinCollateral, 6057 InvalidLimitPrice (**API does zero oracle-side trigger validation — validate client-side: LONG → limit<mark, TP>mark, SL<mark**), 6005 StaleOraclePrice (market closed).

## 3. Market data, prices, streaming
- `GET /v2/tokens` (symbol, mintKey, decimals, isStable, isVirtual, **lazerId**, pythTicker, isToken2022 — 47 tokens live), `GET /v2/prices`, `/v2/prices/{symbol}` — **Pyth Lazer**-sourced, 200 ms updates, fields `{price, exponent, confidence, priceUi, timestampUs, marketSession}`; `marketSession` ∈ `"regular"|"preMarket"|"postMarket"|"overNight"|"closed"` (verified live). Prices are **mainnet only** (querydata.md). The flash-sdk's underlying Lazer proxy default is `https://pyth-lazer-proxy-3.dourolabs.app` (flash-sdk.md) — useful as Throtl's in-ER price cross-check.
- `GET /v2/owner/{owner}` (one-shot) and `GET /v2/owner/{owner}/ws?updateIntervalMs=100..10000` (default 1000): two frame types — `basket` (full snapshot: base64 raw basket bytes + `positionMetrics`/`orderMetrics` maps keyed by marketPubkey, sent on on-chain change) and `metrics` (positions-only oracle ticks, **no orders** — must merge). `MbPositionMetricsDto` gives UI-ready strings (entryPriceUi, sizeUsdUi, pnlWithFeeUsdUi, liquidationPriceUi, leverageUi...). **Warning:** indexer PnL prices exits through `custody.pricing.tradeSpread` (e.g. 10% on SOL) and can read −229% while Flash's own UI shows −4%; compute mark-price PnL client-side: `pnl = (mark−entry)/entry×size×dir − fees`; `liq ≈ entry×(1 ∓ collateral/size×0.92)` (GOTCHAS §20–21). Effective size ≠ collateral×leverage because fills execute at oracle±spread.
- Raw accounts: `/v2/raw/{pools|custodies|markets|baskets}/{pubkey}` (Anchor-deserialized JSON; 92 markets, 1 pool live). v2 balance = `ledger.deposits − basket.debits + basket.pendingCredits` (all cumulative); delegated accounts are invisible to base-chain `getProgramAccounts` — query the **ER RPC**; `UserDepositLedger` zero-copy layout: 852 bytes, `8 disc|1 bump|7 pad|32 owner@16|u32 count@48|20×{32 mint,u64 amount}@52`, discriminator memcmp `9bYPoR9mRKo`@0 (GOTCHAS §18).

## 5. Session keys (the gasless, popup-free path — directly relevant to Throtl's reconciler)
v2 trading requests accept optional `signer` + `sessionToken` (in `MbOpenPositionDto` etc.). **No server endpoint creates sessions** — mint client-side against MagicBlock's session-keys program `KeyspM2ssCJbqUhQ4k7sveSiY4WjnYsrXkC8oDbwde5` via `create_session_v2` (NOT v1): PDA seeds `["session_token_v2", targetProgram(FTv2Rx...), sessionSigner, authority]`, account `SessionTokenV2{authority, target_program, session_signer, fee_payer, valid_until}` (144 bytes), top-up ~0.01 SOL recoverable on `revoke_session_v2` (revoke needs no wallet sig). One wallet approval enables unlimited ER trades signed by the ephemeral key. **Trap:** malformed signer/token fields are silently ignored and the tx falls back to owner-signing. Full flow: https://github.com/flash-trade/examples-v2/blob/main/examples/tap-trade/SESSION-KEYS.md. SDK is TS-only (`@magicblock-labs/gum-sdk` v3, thin Anchor wrapper) — in Dart, hand-build the single `createSessionV2` instruction.

## 6. Builder/referral codes
https://docs.flash.trade/flash-trade/flash-trade-protocol/build-on-flash/partner-referral-program.md: builder codes are **assigned manually by the Flash team** (tier 6, **10% fee rebate**, custom 24 h cap; standard FAF-staking tiers: 20k FAF→2.5% … 1M→7.5%, $1,000/24 h cap), tracked on-chain in the partner's `token_stake` account, rebates accrue as `claimable_rebate_usd`, settle to a `rebate_vault` every 12 h, denominated USD. v1 API exposes `POST /transaction-builder/create-referral` (`{owner, referrer}`) and `init-token-stake`, and v1 open-position takes `privilege`("NONE"/"STAKE"/"REFERRAL"), `tokenStakeFafAccount`, `userReferralAccount`. **Gap: the v2 `MbOpenPositionDto` has NO referral/privilege/builder-code fields** — builder-code attribution on v2 is not yet exposed in the public request shape; you must contact the Flash team (their stated onboarding path anyway) to get a code and confirm v2 wiring.

## 7. Position model & limits
One Basket per wallet; **one position per market**; long and short are separate markets, so max 1 long + 1 short per symbol — `(owner, marketSymbol, side)` is the position identity. Throtl's signed net exposure maps cleanly: increase = open-position, decrease = partial close (mind 97%), flip = reverse (2% haircut) or close+open. Per market+side: **max 5 TP + 5 SL + 5 limit orders** (slots 0–4). Min: $10 collateral on-chain, but **open with ≥$11** or TP/SL placement fails after entry fees (GOTCHAS §14). Max position: $2M on BTC/SOL (collateral-specification.md). Collateral: longs = USDC or same-pool assets (auto-swapped on L1 v1; **SWAP unsupported on v2** — use USDC); **shorts always USDC**. Max leverage ~100x crypto pool (margin-framework.md); maintenance up to 500x crypto. Liquidation: keeper bot run by Flash; trigger orders execute only after price crosses and holds **8–10 s** — Throtl should treat its loss-floor as client/ER-enforced, not rely on Flash SL latency. Fees: SOL/BTC/ETH 0.051% open/close (0.045% discounted); FX 0.030%, metals 0.10%, commodities 0.150%; dynamic hourly borrow (`marginFeePercentage` in quotes).

## 8. Dart/Flutter feasibility
**The HTTP API alone is sufficient — no flash-sdk port needed for trading.** All txs arrive pre-built (the server even patches a known SDK PDA-derivation bug — another reason to use the API); state arrives as UI-ready metric strings; raw basket Borsh decode is optional. Dart needs: (a) v0 VersionedTransaction deserialize/sign preserving existing signatures (`solana` pub.dev package supports v0 messages — verify signature-slot semantics), (b) the hand-built `createSessionV2` ix (§5), (c) manual 852-byte ledger parse + basket debits/pendingCredits for balances via ER `getProgramAccounts`, (d) client-side trigger-price and mark-PnL math (formulas above). Latency: ~30–50 ms is near-validator; expect ~2×RTT floor remotely (~570–620 ms submit→confirmed from a distant region) — use optimistic UI + WS reconcile (GOTCHAS §22).

## 9. Market hours (non-crypto)
https://docs.flash.trade/flash-trade/flash-trade-protocol/perpetuals-specifications/market-hours.md: Crypto 24/7; US equities weekdays 9:30 AM–4 PM ET (closed weekends/US holidays); FX & metals Sun 5 PM→Fri 5 PM ET with a daily 60-min break at 5 PM ET; commodities Sun 6 PM→Fri 5 PM ET; metals/commodities follow CME holidays. Closed markets → `marketSession:"closed"`, 503 "price missing", on-chain 6005 StaleOraclePrice. Throtl must gate the thumb control per-symbol on `marketSession`.
